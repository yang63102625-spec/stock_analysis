# 股票分析策略介绍

> 面向同事的简要说明，帮助快速了解本系统的分析逻辑与策略体系。

---

## 一、核心交易理念（8 条）

所有分析策略均基于以下理念，AI 在生成报告时会严格遵守：

| 编号 | 理念 | 简要说明 |
| ---- | ---- | ---- |
| 1 | **严进策略** | 不追高，乖离率 > 动态阈值不买入；ATR自适应（高波动放宽、低波动收紧） |
| 2 | **趋势交易** | MA5 > MA10 > MA20 多头排列才考虑；空头排列不碰 |
| 3 | **效率优先** | 关注筹码集中度、量能配合，确认趋势有效性 |
| 4 | **买点偏好** | 优先缩量回踩 MA5/MA10 支撑，跌破 MA20 观望 |
| 5 | **风险排查** | 减持、业绩预亏、监管处罚等利空一票否决 |
| 6 | **量价配合** | 多日量能趋势验证（3/10/20日量比）、天量预警、量能衰竭检测 |
| 7 | **资金验证** | 主力资金+北向资金趋势共振确认，资金面三角验证 |
| 8 | **强势放宽** | 龙头股、强势趋势股可适当放宽乖离率要求 |

## 二、内置分析策略（13 种）

策略 id 与 YAML 中 `name` 一致，用于 `AGENT_SKILLS` 和 `/ask` 命令。

### 趋势类（trend）

| id | 策略 | 说明 | 适用场景 |
| -- | ---- | ---- | ---- |
| bull_trend | 多头趋势 | 识别 MA5>MA10>MA20 多头排列、回踩低吸机会 | 常规个股分析默认策略 |
| ma_golden_cross | 均线金叉 | 检测 MA5 上穿 MA10/MA20，配合量能确认 | 趋势转折初期 |
| volume_breakout | 放量突破 | 放量突破阻力位，区分真突破与假突破 | 趋势加速阶段 |
| shrink_pullback | 缩量回踩 | 上升趋势中缩量回踩均线企稳反弹 | 理想入场点 |
| dragon_head | 龙头策略 | 板块启动时率先领涨的个股 | 板块轮动、进攻环境 |

### 形态类（pattern）

| id | 策略 | 说明 | 适用场景 |
| -- | ---- | ---- | ---- |
| one_yang_three_yin | 一阳穿多阴 | 大阳后 2~4 根小阴缩量整理，再阳线收复 | 洗盘结束、趋势延续 |

### 反转类（reversal）

| id | 策略 | 说明 | 适用场景 |
| -- | ---- | ---- | ---- |
| bottom_volume | 底部放量 | 长期下跌后底部异常放量，可能主力入场 | 反转信号，仓位需严格控制 |

### 框架类（framework）

| id | 策略 | 说明 | 适用场景 |
| -- | ---- | ---- | ---- |
| box_oscillation | 箱体震荡 | 箱底买入、箱顶减仓，区间波段操作 | 震荡市 |
| chan_theory | 缠论 | 中枢、背驰、买卖点，简化为可量化判定 | 需一定缠论基础 |
| wave_theory | 波浪理论 | 5 浪推进 + 3 浪调整，斐波那契目标价 | 中长线结构分析 |
| emotion_cycle | 情绪周期 | 恐慌底部布局、狂热顶部离场，逆向操作 | 情绪极端点 |
| multi_factor | 多因子复合 | 趋势+量能+形态+情绪四维评分，交叉验证 | 降低单一策略误判 |
| risk_management | 统一风控框架 | 止损、仓位、风险过滤，优先级高于买入信号 | 所有策略通用 |

## 三、分析输出：决策仪表盘

每只股票的分析报告采用**决策仪表盘**格式，包含：

- **核心结论**：一句话该买该卖、空仓/持仓建议
- **数据透视**：均线、乖离率、量比、筹码结构、资金面
- **舆情情报**：最新消息、风险点、利好催化
- **作战计划**：理想买入点、阶梯止盈（5%/10%）、止损位（MA20/7%）、仓位建议

### 评分体系（7维度满分 100，2026-05 重平衡）

| 维度 | 权重 | 说明 |
| ---- | ---- | ---- |
| 趋势 | 30 | MA多头排列、均线斜率、价格位置 |
| 量能 | 18 | 3/10/20日量比、天量预警(>5倍)、量能衰竭检测；市况感知（牛市18/震荡10/熊市0）；加速期放量上涨降权 14→6；强势趋势缩量上涨降权 7→3 |
| 乖离率 | 15 | 阶段化阈值 + ATR(20)动态调整，区分MA20方向 |
| MACD | 13 | 金叉/死叉、柱状线方向、零轴位置（上调 10→13） |
| 资金面 | 13 | 主力资金 + 北向资金，外部仍 0-10 raw、内部 ×1.3 加权（上调 10→13） |
| 支撑 | 6 | MA10 支撑(5) + MA20 趋势完整性(1)；移除与乖离率重复的 MA5 支撑 |
| RSI | 5 | 超买超卖（含2日企稳确认） |

**估值惩罚**（不计入维度，独立扣分）：PE>200 -15、PE>100 -8、亏损股 -5。

**Bias 死区修复**：MA20 数据缺失时改用线性插值兜底，避免阶梯断档。

### 大盘环境修正系数

基于上证指数 MA20 方向对最终评分进行修正，避免在系统性下跌环境中给出高分误导：

| 环境等级 | 条件 | 修正系数 | 评分硬上限 | BUY/STRONG_BUY 阈值 |
| -------- | ---- | -------- | ---------- | -------------------- |
| strong_bull | 沪指 > MA20 且 MA20 上行 | ×1.05 | — | 60 / 75 |
| bull | 沪指 > MA20 | ×1.00 | — | 60 / 75 |
| neutral | 沪指 ≈ MA20（缓冲区） | ×1.00 | — | 60 / 75 |
| bear | 沪指 < MA20 | ×0.85 | 75 | **70 / 85**（阈值 +10） |
| strong_bear | 沪指 < MA20 且 MA20 下行 | ×0.75 | **60** | **70 / 85**（阈值 +10） |

**双重防御**：熊市同时启用乘数缩放、评分硬上限和分类器阈值上调，防止"完美个股"在熊市误触发 BUY 信号。

最终评分 = 7维度裸分合计 × 修正系数

### 乖离率阶段化规则

不同趋势阶段对乖离率容忍度不同，阈值采用分阶段设定 + ATR(20) 动态微调：

| 阶段 | 基准阈值 | 说明 |
| ---- | -------- | ---- |
| 启动期 | 6.0% | 刚突破平台，允许较大乖离 |
| 主升期 | 5.0% | 趋势加速，正常乖离容忍 |
| 加速期 | 3.5% | 末端加速风险高，严格限制 |

**ATR 动态调整**：基准阈值 × (ATR_20 / 收盘价) 比例系数，高波动股放宽、低波动股收紧。当 ATR 无法从数据源获取时使用本地计算兜底。

### 资金面评分细则

| 资金类型 | 净流入额 | 得分 |
| -------- | -------- | ---- |
| 主力资金 | ≥ 5000万 | 6 |
| 主力资金 | ≥ 1000万 | 3 |
| 主力资金 | ≥ 100万 | 1 |
| 主力资金 | < 100万 | 0 |
| 北向资金 | ≥ 3000万 | 4 |
| 北向资金 | ≥ 500万 | 2 |
| 北向资金 | ≥ 100万 | 1 |
| 北向资金 | < 100万 | 0 |

数据源：Tushare moneyflow（个股主力资金 + 北向资金），门槛 100万过滤噪音。

### 量能异常惩罚

- **量能衰竭**：近日成交量持续萎缩至极低水平 → 扣 5 分
- **天量预警**：当日量 > 20日均量 × 5 → 扣 15 分（主力出货风险）

### 止盈止损一致性约束（统一交易点位引擎）

picker / analyzer / backtest 三层共用 `src/services/trade_levels.py` 计算交易点位，消除以往"三套规则"的不一致：

- **止盈策略（阶梯 + Trailing）**：
  - 浮盈 6%：减仓 1/3，止损上移至成本
  - 浮盈 12%：再减 1/3，止损上移至 +6%
  - 浮盈 ≥15%：剩余仓位启用 trailing — 跌破 MA10 OR 回撤 ATR×2.5 才卖出
  - 反转策略例外：浮盈 +15% 减半，+20% 全清（让真反转走完主升段）
  - 尾盘买入策略例外：次日尾盘前必须清仓
- **止损规则**（按市值差异化）：小盘 -6% / 中盘 -6% / 大盘 -7%，同时考虑 MA20 技术止损
- **仓位规则**：小盘 5% / 中盘 12% / 大盘 15% 基线；R/R≥3 时 ×1.30 cap 22%
- **盈亏比硬过滤**：R/R < 2.0 的候选直接剔除（picker 层）
- **多策略共振加分**：2 命中 +8、3 命中 +25 并强制纳入 AI 精选
- **行业集中度上限**：同 SW 一级行业最多保留 N 只（默认 2，env `PICKER_INDUSTRY_TOP_N`）
- **市场环境感知仓位**：弱市 ×0.6 / 中性 ×0.85 / 强市 ×1.0 自动缩放
- **回测 = 生产**：picker_backtest 用同一引擎，加 0.3% 滑点 + 涨停日不可成交过滤
- LLM 必须直接采用 `trade_levels` 字段中的数值，**禁止**自行计算止盈止损

## 四、使用方式

### 默认策略组合

系统默认启用 4 个策略：`bull_trend`、`ma_golden_cross`、`volume_breakout`、`shrink_pullback`。

### 自定义策略

通过环境变量 `AGENT_SKILLS` 指定，逗号分隔：

```env
AGENT_SKILLS=bull_trend,ma_golden_cross,shrink_pullback
```

### Web / Bot 指定策略

- **Web 对话页**：可选择不同策略进行分析
- **Bot 命令**：`/ask 600519 用缠论分析` 或 `/ask 600519 chan_theory`

## 五、策略与市场环境

多数策略会根据**大盘环境**调整信号强度。系统基于上证指数与 MA20 的关系自动判定当前环境等级，并施加评分修正系数：

| 环境 | 判定条件 | 修正系数 | 策略影响 |
| ---- | ---- | ---- | ---- |
| **强多头 (strong_bull)** | 沪指 > MA20 且 MA20 方向上行 | ×1.05 | 信号正常，评分加成 |
| **多头 (bull)** | 沪指 > MA20 | ×1.00 | 信号正常执行 |
| **中性 (neutral)** | 沪指 ≈ MA20 缓冲区 | ×1.00 | 仓位适度、部分策略仅接受高级别信号 |
| **空头 (bear)** | 沪指 < MA20 | ×0.90 | 评分打9折，仓位减半 |
| **强空头 (strong_bear)** | 沪指 < MA20 且 MA20 下行 | ×0.85 | 评分打85折，多数策略暂停 |

量能评分也随市况动态调整：牛市缩量回踩满分18、震荡市10分、熊市0分（缩量无效）。

## 六、AI 智能选股

每日分析后可运行 **AI 智能选股**，从全市场筛选优质标的并推送推荐。

> 量化筛选策略的详细参数与流程见 [选股策略详解](picker-strategies-guide.md)。

### 两阶段流程

1. **量化筛选**：从 5000+ 只股票中，按策略（买回踩/突破/底部反转）并行筛选，各策略 Top30 合并去重
2. **后过滤**：乖离率、连板、连涨、健康回踩（策略参数）、B 浪风险（可选）
3. **AI 精选**：结合大盘、板块、新闻，由 LLM 精选 1–5 只并给出理由（宁缺毋滥）

### 选股原则（与核心理念一致）

- **严进策略**：量化层排除乖离率 > 8%；AI 层乖离率 < 3% 优先，5–8% 降级
- **趋势质量**：60 日涨幅（>30% 衰减，避免买在末端）、今日涨幅、量比 >1.0、换手率；今日涨幅 > 9% 排除（追高风险）
- **估值安全**：PE 10–30 倍理想，> 50 倍谨慎；量化层排除 PE > 100
- **板块共振**：个股与领涨板块重合时加分
- **行业分散**：建议推荐标的分散于不同行业，避免单行业过度集中
- **筹码集中度**：AI 精选阶段补充筹码数据（90%集中度、获利比例），筹码高度集中加分
- **空仓触发**：池中乖离率 >5% 占比 >60% 时，建议空仓观望

### 数据层说明

- **Tushare 路径**：通过 `trade_cal` 获取 60 个交易日前的日期，拉取日线计算 60 日涨跌幅，确保动量过滤与打分生效
- **AkShare 路径**：实时行情已含 60 日涨跌幅，直接使用

### 配置与触发

- 环境变量：`PICKER_ENABLED=true`（默认）启用每日分析后自动选股
- 命令行：`python main.py --picker` 完整分析后选股；`python main.py --picker-only` 仅运行选股（快速验证）
- 测试：`./test.sh picker` 完整选股（需网络）；`./test.sh picker-validation` 策略验证（离线）
- API：`POST /api/v1/picker/recommend`

## 八、适用范围与局限

| 项目 | 说明 |
| ---- | ---- |
| **适用市场** | A 股（沪深主板、创业板、科创板）；港股、美股需单独适配 |
| **数据质量** | 依赖 Tushare / AkShare / efinance 等数据源，非交易日或数据延迟会影响结果 |
| **60 日涨跌幅** | Tushare 路径通过 trade_cal + daily 计算；AkShare 实时行情自带 |
| **乖离率计算** | 选股器需拉取日线计算 MA5；若无日线数据，该层过滤会跳过 |
| **LLM 主观性** | AI 精选依赖大模型，同一池子不同模型/温度可能给出不同推荐 |
| **非投资建议** | 本系统仅供学习与研究，不构成任何投资建议，请独立判断并自担风险 |
